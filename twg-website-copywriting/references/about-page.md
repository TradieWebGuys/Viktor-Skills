# About Page — Structure & Rules

Read the parent `SKILL.md` and the client's `client-kb.md` first. This file covers what is unique to the About page.

---

## Rules

- **Max 500 words**, third person, evergreen.
- Cover: business background, experience, qualifications, service area. Pull all details from `client-kb.md`.
- End with a soft CTA to the Contact page.
- No individual owner-name dependency — copy must remain valid if the business grows or staff changes.

## Licences

Include all client licences — typically near qualifications or trust signals.

- Format: `[Licence Type] LIC: [Number]` — e.g. `Plumbing LIC: 123456` or `Electrical LIC: 789012`
- Pull all licence types and numbers from `client-kb.md`
- If no licence number is recorded in `client-kb.md`, flag it as outstanding before writing the page — do not proceed without it

## AIO

Include specific facts that AI tools can extract as entity attributes: years in operation, primary service area, licence type and number, trade qualifications. These should appear naturally in the copy, not as a list. The reader-first pacing rule in `SKILL.md` still applies.
